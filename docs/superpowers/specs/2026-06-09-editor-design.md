# Web 16 灰阶图片编辑器设计

## 背景

本项目是 ESP-IDF 的 ESP32-S3 电子纸日历工程，当前默认屏幕为 ED060KD1，并在固件中使用 `EPD_ROT_PORTRAIT`。编辑器作为仓库内独立工具放在 `editor/`，不牵动固件构建系统。

编辑器目标是提供一个所见即所得的 16 灰阶图片编辑工具，用于为电子纸屏幕制作图片资源，并导出固件可消费的二进制数据或便于检查的 BMP。

## 范围

第一版实现以下功能：

- 选择屏幕大小，默认提供 `ED060KD1 portrait 1072 x 1448`，并支持自定义宽高。
- 导入 SVG、PNG、JPG 图片，并调整位置和大小。
- 绘制矩形、圆、直线、多边形。
- 添加后的所有元素都可以继续编辑。
- 保存项目为自包含 JSON，并从 JSON 导入项目。
- 导出 `bin`、`raw`、`bmp` 三种格式。

第一版不实现以下功能：

- 元素旋转。
- 贝塞尔路径编辑。
- 图形布尔运算。
- 固件内嵌 Web 服务。
- npm、Vite、React 或其它前端构建链。

## 架构

编辑器放在 `editor/`，采用纯静态 Vanilla JS：

- `editor/index.html`：三栏工作台结构，包含顶部项目和导出操作、左侧工具区、中央画布区、右侧属性和图层区。
- `editor/styles.css`：桌面式工具界面样式，重点支持大尺寸竖屏画布的缩放查看。
- `editor/app.js`：应用状态、事件处理、绘制、导入导出、项目序列化。
- `editor/README.md`：使用方式、支持格式、导出语义、项目 JSON 结构说明。

第一版不拆分 ES modules，保证直接打开 `editor/index.html` 时也能运行。如果某些浏览器对本地文件的图片解码策略有差异，用户可以用任意静态服务打开 `editor/`。

## 界面布局

采用三栏工作台：

- 左侧：屏幕尺寸、导入图片、绘制工具、项目导入。
- 中央：可缩放的编辑画布，显示 16 灰阶量化后的结果，并提供选择框和缩放手柄。
- 右侧：选中元素属性、图层列表、图层顺序、可见性、锁定、删除。

三栏布局优先保证图层和属性编辑空间，适合反复调整多个可编辑元素。

## 数据模型

项目文件是自包含 JSON，保存画布尺寸、屏幕预设和所有元素。导入图片保存为 data URL，因此项目文件不依赖原始图片路径。

顶层结构：

```json
{
  "version": 1,
  "screenPreset": "ED060KD1 portrait",
  "canvas": {
    "width": 1072,
    "height": 1448
  },
  "layers": []
}
```

公共图层字段：

- `id`：稳定 ID。
- `type`：`image`、`rect`、`ellipse`、`line`、`polygon`。
- `name`：图层名称。
- `visible`：是否显示。
- `locked`：是否锁定。
- `x`、`y`、`width`、`height`：元素边界。

图片层额外字段：

- `src`：图片 data URL。
- `sourceWidth`、`sourceHeight`：原始图片尺寸。
- `opacity`：透明度。

图形层额外字段：

- `fillGray`：填充灰阶，范围 `0..15`。
- `strokeGray`：描边灰阶，范围 `0..15`。
- `strokeWidth`：描边宽度。
- `points`：直线和多边形使用的点数据。

灰阶约定：

- `0 = 黑`
- `15 = 白`

## 渲染流程

渲染统一通过隐藏 canvas 和主画布完成：

1. 根据项目尺寸创建隐藏 canvas。
2. 用白底清空隐藏 canvas。
3. 按图层顺序绘制所有可见元素。
4. 将隐藏 canvas 的 RGBA 像素转换为亮度。
5. 将亮度量化到 `0..15` 的 16 灰阶。
6. 把量化结果映射回主画布显示。

导入图片、形状绘制和导出都使用同一份量化像素数据，确保所见即所得。

## 编辑交互

第一版支持以下编辑能力：

- 选择元素。
- 拖动元素。
- 通过角点缩放元素。
- 在右侧属性区精确编辑 `x`、`y`、`width`、`height`。
- 修改图形填充灰阶、描边灰阶、线宽。
- 修改图片透明度。
- 修改图层名称、可见性、锁定状态。
- 调整图层顺序。
- 删除元素。

添加图片时，编辑器按画布尺寸自动适配缩放，并居中放置。

多边形第一版使用点击画布添加点，双击或按钮结束绘制。

## 导出格式

导出前统一得到 `width * height` 的 16 灰阶像素数组，每个像素值范围 `0..15`。

### bin

`bin` 是纯 4bpp packed pixels：

- 按行从左到右、从上到下。
- 每字节保存两个像素。
- 高 4 bit 是左侧或前一个像素。
- 低 4 bit 是右侧或后一个像素。
- 如果画布宽度是奇数，每行最后一个像素放高 4 bit，低 4 bit 补 `0`。

文件长度：

```text
ceil(width / 2) * height
```

默认 ED060KD1 竖屏 `1072 x 1448` 的 `bin` 长度为 `776128` 字节。

### raw

`raw` 第一版定义为 epdiy 风格 framebuffer 数据，但具体字节布局先与 `bin` 保持一致，即同样的 4bpp 行序打包。

README 必须明确说明此约定。如果后续固件端确认需要 epdiy 内部 nibble 取反、行对齐或其它布局规则，再单独调整 `raw` 导出。

### bmp

`bmp` 是标准 8-bit 灰度 BMP：

- 使用 256 项灰度调色板。
- 16 灰阶像素映射为 `0, 17, 34, ..., 255`。
- 像素行按 BMP 规范写入，并处理 4 字节行对齐。

BMP 用于电脑预览和校验，不作为固件优先读取格式。

## 错误处理

- 导入图片只接受 `image/svg+xml`、`image/png`、`image/jpeg`。
- 图片解码失败时显示错误，不改变当前项目。
- 项目 JSON 解析失败时显示错误，不改变当前项目。
- 项目版本不支持时显示错误，不改变当前项目。
- 自定义画布宽高必须是正整数，并设置上限以避免创建过大 canvas。
- 导出空白白底画布是合法操作。
- 项目导入成功前不覆盖当前状态，成功后一次性替换项目。

## 验证

由于这是强 UI 工具，第一版不为布局和鼠标交互硬套单元测试。验证以浏览器手动流程和导出长度检查为主。

实现完成后需要验证：

1. 使用本地静态服务打开 `editor/`。
2. 选择默认 ED060KD1 竖屏尺寸。
3. 导入 PNG、SVG、JPG。
4. 添加矩形、圆、直线、多边形。
5. 拖动和缩放元素。
6. 修改属性和图层顺序。
7. 保存项目 JSON。
8. 重新导入项目 JSON，并确认画面和图层恢复。
9. 导出 `bin`、`raw`、`bmp`。
10. 检查默认尺寸下 `bin/raw` 文件长度是 `776128` 字节。

如果存在本地浏览器自动化能力，应补充一次页面加载和关键按钮可见性检查；若不可用，记录未自动化验证的原因。

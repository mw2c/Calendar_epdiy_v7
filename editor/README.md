# EPD 16 灰阶图片编辑器

`editor/` 是一个无构建依赖的静态 Web 工具，用于制作电子纸 16 灰阶图片资源。

## 打开方式

可以直接用浏览器打开：

```text
editor/index.html
```

如果浏览器对本地文件的图片解码策略有限制，可以在仓库根目录运行静态服务：

```sh
python3 -m http.server 8765 --directory editor
```

然后打开：

```text
http://localhost:8765
```

## 画布尺寸

默认屏幕预设：

```text
ED060KD1 portrait 1072 x 1448
```

也可以选择自定义宽高。宽高必须是正整数，单边最大值为 `4096`。

## 支持导入

支持导入：

- SVG
- PNG
- JPG/JPEG

导入图片会创建图片图层，并按画布 80% 范围内等比缩放、居中放置。图片图层可以继续移动、缩放和调整透明度。

## 绘制与编辑

支持以下元素：

- 矩形
- 圆/椭圆
- 直线
- 多边形
- 导入图片

添加后的元素可以选择、拖动、缩放、修改属性、调整图层顺序、隐藏、锁定或删除。第一版不支持旋转和复杂路径编辑。

## 项目文件

项目保存为自包含 JSON：

- `version`
- `screenPreset`
- `canvas.width`
- `canvas.height`
- `layers`

导入图片会以 data URL 存入 JSON，因此项目文件不依赖原始图片路径。

## 灰阶约定

内部像素值范围为 `0..15`：

```text
0  = 黑
15 = 白
```

编辑器显示和导出都使用同一份 16 灰阶量化结果。

## 导出格式

### BIN

`bin` 是纯 4bpp packed pixels：

- 按行从左到右、从上到下。
- 每字节保存两个像素。
- 高 4 bit 是左侧或前一个像素。
- 低 4 bit 是右侧或后一个像素。
- 如果画布宽度是奇数，每行最后一个像素放高 4 bit，低 4 bit 补 `0`。

文件长度：

```text
ceil(width / 2) * height
```

默认 `1072 x 1448` 的长度是：

```text
776128 bytes
```

### RAW

第一版 `raw` 与 `bin` 使用相同的 4bpp 行序打包布局。后续如果固件读取端确认需要 epdiy 内部 nibble 取反、行对齐或其它 framebuffer 布局，再单独调整 `raw` 导出。

### BMP

`bmp` 是标准 8-bit 灰度 BMP：

- 使用 256 项灰度调色板。
- 16 灰阶映射为 `0, 17, 34, ..., 255`。
- BMP 行按 4 字节对齐并自底向上写入。

BMP 主要用于电脑预览和校验。
